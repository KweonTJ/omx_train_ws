# PPO 재학습 기록

## 재학습 목적

실제 로봇 bringup의 Stay 자세와 기존 PPO 학습 시작 자세가 달라 첫 정책 명령에서 큰 관절 오차와 토크 충격이 발생할 수 있었다. 이번 학습은 실제 Stay를 MuJoCo keyframe, reset, 복귀 목표, 정책 metadata에 동일하게 적용한 새 버전을 만드는 작업이다.

| 항목 | 기존 v2 | 재학습 v3 |
|---|---|---|
| 정책 버전 | `arm_delivery_residual_v2` | `arm_delivery_residual_v3_robot_stay` |
| Stay 관절 | `[0, 0, 1.38, -1.38] rad` | `[0.104311, 0.027612, -0.001534, -1.638291] rad` |
| 정책 시작 상태 | 학습 전용 자세 | 실제 `hardware.launch.py` 자세 |
| 누적 PPO step | `537,088` | `660,000` |
| 학습 장치 | CPU 중심 | CUDA, RTX 4050 Laptop GPU |

## 반영 내용

- `scene.xml`의 `stay` keyframe과 actuator control을 실제 로봇 자세로 변경했다.
- metadata가 `key_qpos[:4]`로 바퀴 관절을 잘못 읽던 문제를 고치고 `joint1`~`joint4` 이름으로 qpos 주소를 조회한다.
- 파지된 상태로 시작하는 episode를 고정 관절값이 아니라 매 물체 좌표의 pregrasp IK 해에서 초기화한다.
- 타워 끝단 접촉 회귀 시험을 추가하고 EEF pregrasp 높이를 `25.0 mm`에서 `22.5 mm`로 조정했다.
- 12단계 curriculum의 각 단계에서 50회 평가 최고 체크포인트만 다음 단계로 승격했다.

## 판정

v3는 실제 Stay 자세 계약을 만족하고 전체 Sim2Real 목표 `>= 90%`를 통과했으므로 실기 저속 시험 후보로 채택한다. 이는 실기 최종 승인 판정이 아니다. 실패는 대부분 로봇 기준 양의 X 끝단에 집중됐고 높이 변동 단독 성능도 v2보다 낮다.

실기 적용은 다음 조건을 유지한다.

1. bringup이 실제 Stay로 이동한 뒤 RL 노드를 시작한다.
2. 첫 시험은 중앙 위치, 저속 Pick과 Stay 복귀까지만 수행한다.
3. 양의 X 끝단 목표는 제한하거나 베이스 위치를 다시 맞춘다.
4. E-Stop 준비 상태에서 높이별 반복 시험을 별도로 기록한다.
5. 충돌 0회 기준을 통과하기 전 전체 배송 자동 반복을 승인하지 않는다.
